# cowork-to-code-bridge

Run commands on your Mac from inside a [Claude Cowork](https://claude.ai/cowork) sandbox.

Cowork can't reach your local environment — no shell, no Docker, no SSH keys, no `~/.claude.json`. This bridge is a ~400-line file-based RPC that fixes that, with one paste per side.

## How it works

```
  Cowork sandbox                                Mac
  ──────────────                                ───
  writes JSON →   bridge/queue/cmd_*.json  ← polled by daemon
                                                  ↓ runs whitelisted script
                                              ~/.cowork-to-code-bridge/scripts/
                                                  ↓
  reads JSON ←   bridge/results/cmd_*.json ← daemon writes
```

No network listeners. No long-lived processes inside Cowork. Just files on a shared bind-mount, a tiny launchd-managed daemon on the Mac, and a Python client.

## Install

### 1. On your Mac (one time, ~30 seconds)

```bash
curl -fsSL https://raw.githubusercontent.com/abhinaykrupa/cowork-to-code-bridge/main/install.sh | bash
```

This:
- pip-installs the Python package
- creates `~/.cowork-to-code-bridge/` with a fresh security token
- installs a launchd plist so the daemon auto-restarts on login
- starts the daemon now

### 2. In any Cowork session

Paste this:

```
Install the cowork-to-code-bridge plugin from https://github.com/abhinaykrupa/cowork-to-code-bridge
```

Cowork will detect the Mac daemon, install the plugin side, and confirm the bridge is live. After that you can ask things like:

- *"run `git status` on my Mac"*
- *"run `pytest tests/` on my project"*
- *"run my deploy script on my Mac"*

## Security model

| Layer | Mechanism |
|---|---|
| Authentication | Shared token (`BRIDGE_TOKEN`) generated by `install.sh`, stored in `~/.cowork-to-code-bridge/.env` with `chmod 600`. Every command must include it. |
| Authorization | Daemon only runs scripts in `~/.cowork-to-code-bridge/scripts/`. No arbitrary shell. |
| Path traversal | Script names must match `^scripts/[A-Za-z0-9_/.-]+\.(sh\|py)$`. Resolved paths verified to stay under the scripts dir. |
| Resource limits | Per-command timeout, capped at `BRIDGE_MAX_TIMEOUT` (default 600s). Output truncated to 64 KiB. |
| Process isolation | Daemon runs as your user; subprocess inherits its env minus the secret-loading rules. No privilege escalation. |

What this **doesn't** protect against:

- A malicious script you whitelist yourself (you wrote it, you own it).
- Anything that bypasses the bridge — e.g. someone with write access to the bind-mount itself can submit commands directly.

## Adding your own scripts

```bash
# On your Mac
cat > ~/.cowork-to-code-bridge/scripts/my_thing.sh <<'EOF'
#!/usr/bin/env bash
# your script here
echo "did the thing with args: $*"
EOF
chmod +x ~/.cowork-to-code-bridge/scripts/my_thing.sh
```

From Cowork:

```python
from cowork_to_code_bridge import call_remote
r = call_remote("scripts/my_thing.sh", args=["foo", "bar"], timeout=30)
print(r["stdout"], r["exit_code"])
```

## Why not MCP?

| | MCP | This bridge |
|---|---|---|
| Transport | JSON-RPC over stdio/HTTP/SSE | File rename on a shared volume |
| Server process | Long-lived per session | One launchd daemon for the whole Mac |
| Cowork compatibility | Limited — sandbox can't launch processes | Works because the daemon is on the Mac, not in Cowork |
| Setup complexity | Server config + tool definitions + transport choice | One `curl \| bash` + one paste |
| Fits when | You want structured tool-calling with rich schemas | You want to shell out to your Mac and get the output back |

The bridge is closer to a mailbox than an RPC server. It's not trying to compete with MCP — it's solving a problem MCP doesn't fit (Cowork sandbox can't reach your Mac).

## Uninstall

```bash
# On your Mac
bash <(curl -fsSL https://raw.githubusercontent.com/abhinaykrupa/cowork-to-code-bridge/main/daemon/uninstall.sh)
python3 -m pip uninstall cowork-to-code-bridge
```

## Development

```bash
git clone https://github.com/abhinaykrupa/cowork-to-code-bridge
cd cowork-to-code-bridge
python3 -m pip install -e ".[dev]"

# Run daemon in foreground for testing
BRIDGE_ALLOW_UNAUTH=1 python3 -m cowork_to_code_bridge.daemon

# In another shell, drive it
python3 -c "from cowork_to_code_bridge import call_remote; print(call_remote('scripts/ping.sh'))"
```
